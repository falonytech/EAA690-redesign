fix: calendar page now reflects published Sanity events immediately

Two bugs prevented newly published events from appearing on /calendar:

1. Missing `export const dynamic = 'force-dynamic'` — Next.js was treating
   the page as static and baking it as HTML at build time. New events published
   after deployment were invisible until the next Vercel deploy.

2. getAllEvents() used useCdn:true in production — Sanity's CDN edge cache
   could lag several minutes behind a freshly published document. Fixed by
   using a dedicated no-CDN client for this query so events appear immediately
   after publishing via /admin/content.
